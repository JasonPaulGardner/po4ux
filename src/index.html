Web Performance Optimization

for Improved User Experience

What is WPO?

Web performance optimization is the process by which individual aspects of web pages are optimized in order to achieve greater overall performance.

Why Worry About Performance

Yahoo Says [1]
Adding 400ms to the homepage saw a 5 – 9% drop in full-page traffic
Google found [2]
Adding a 400ms delay to the search results caused a 0.76% drop in searches per user.
Bing measured [3]
a 2.8% drop in revenue with a 1 second delay.

Why Worry About Performance

Shopzilla claim [4]
Building a better performing website (speed improvements)
  • increased revenue by 1-12%
  • increased page views by 25%
  • reduced hardware costs by 50%

Why Worry About Performance

The Aberdeen Group discovered [5]
A one second delay equates to an average 7% reduction in conversions.
Forrester reports [6]
Users expect pages to load in under two seconds.

What is taking so long

The Front-End Usual Suspects

  • Too many connections
  • Too many KB's
  • Unnecessary data transfers
  • Page complexity
  • Poor page render order
  • Scaled images

Questions?

Gzip Components

  • Approximately 90% of today's Internet traffic travels through browsers that claim to support gzip.
  • Gzipping generally reduces the response size by about 70%.

Add an Expires or a Cache-Control Header

For static components:
Implement "Never expire" policy by setting far future Expires header
For dynamic components:
Use an appropriate Cache-Control header to help the browser with conditional requests

Minimize HTTP Requests

  • Combine all scripts into a single script file.
  • Combine all CSS into a single stylesheet.
  • Use Image Sprites.
  • Use Data Urls.

Use a Content Delivery Network

Shorter delivery routes
direct request to the nearest server
Benefit from cross site caching
source common javascript files (jQuery) from shared CDN (either Google or Microsoft)
CDNs offer other advantages
  • Higher capacity and scale
  • Higher Availability
  • Lower delivery cost
  • Reduced network load

Put Stylesheets at the Top

  • Allows progressive render
  • Prevents redrawing of elements

Use Script loading best practises

Put Script(s) at the Bottom
Minimize the number of
  • Resources that are blocked from downloading
  • Elements that are blocked from rendering
because they are below the script.
Async Javascript
  • Removes Frontend Single Point of Failure

Make JavaScript and CSS External

  • Maximises cache benifit

Minify JavaScript and CSS

  • Even if you gzip your scripts and styles, minifying them will still reduce the size by 5% or more.

Avoid Redirects

  • Avoid valueless RTT and target the resource directly

Use Cookie-free Domains

Reduces request size
upload is more expensive than download
If cookies are unavoidable minimize their impact on the UX by
  • Eliminating unnecessary cookies.
  • Keeping cookie sizes as low as possible.
  • Setting cookies at the appropriate domain level so other sub-domains are not affected
  • Setting an appropriate Expires date.

Reduce DNS Lookups

DNS has a cost
It typically takes 20-120 milliseconds for DNS to lookup the IP address for a given hostname.

Remove Duplicate Scripts

This isn’t as unusual as you might think. A review of the ten top U.S. web sites shows that two of them contain a duplicated script.

Content management system and web frameworks are the usual cause of duplicate script tags. Be aware of the issue and consider implementing a script management system.

Other Things to consider

  • Configure / remove Etags
  • Flush buffer early
  • Keep request to one packet in size (1500 bytes)
  • Use GET for AJAX request
  • Post-load companents
  • Preload components
  • Reduce the number of Dom elements
  • Minimize the number of iframes
  • No 404s
  • Minimize Dom access
  • Remove unused CSS
  • Use efficient CSS selectors
  • Pack components into a multipart document
